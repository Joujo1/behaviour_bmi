# Connecting a New ItsyBitsy MCU

The ItsyBitsy M4 Express (SAMD51) is the audio peripheral for the click stimulus. It receives TTL trigger pulses from the Pi and plays back the click waveform through its two DAC channels. The firmware is in [itsybitsy_click_player/click_player/click_player.ino](../../itsybitsy_click_player/click_player/click_player.ino).

---

## How it works

The Pi fires a 100 µs TTL HIGH pulse on BCM GPIO 9 (right) or BCM GPIO 10 (left) at each scheduled click time. The ItsyBitsy detects the rising edge via interrupt and immediately resets the DAC playback position to zero. A TC3 timer ISR running at 192 kHz clocks out one 12-bit sample per channel per period, playing the pre-baked 3 ms waveform from the `CLICK_WAV` array. When both channels finish, the timer stops to eliminate DAC switching noise during silence.

Pinout on the ItsyBitsy:

| ItsyBitsy pin | Connected to | Function |
|---|---|---|
| D9 | Pi BCM GPIO 10 | Left trigger input |
| D10 | Pi BCM GPIO 9 | Right trigger input |
| A0 | Left amplifier (via 10 µF cap) | Left DAC output |
| A1 | Right amplifier (via 10 µF cap) | Right DAC output |

---

## Flashing the firmware

1. Connect the ItsyBitsy to any computer via USB.
2. Open `itsybitsy_click_player/click_player/click_player.ino` in the Arduino IDE.
3. Install the **Adafruit SAMD** board package if not already installed (Boards Manager → search "Adafruit SAMD").
4. Select board: **Adafruit ItsyBitsy M4 (SAMD51)**.
5. Click **Upload**.

The ItsyBitsy does not need any libraries beyond the Arduino core — the firmware accesses hardware registers directly.

## Changing the click waveform

The waveform is a pre-baked array of 576 twelve-bit unsigned samples (`CLICK_WAV`) embedded in the `.ino` file. It is generated by [itsybitsy_click_player/run_once_generate_waveform.py](../../itsybitsy_click_player/run_once_generate_waveform.py).

Current waveform parameters:

| Parameter | Value | Description |
|---|---|---|
| Sample rate | 192 000 Hz | TC3 timer frequency |
| Click duration | 3 ms (576 samples) | `WIDTH` in the generator |
| Tones | 2, 4, 8, 16 kHz | Summed pure tones |
| Attenuation | 40 dB per tone | Keeps peak level sane |
| Fade | 2 ms cosine² taper on each end | Prevents onset/offset clicks |

To change the waveform:

1. Edit the parameters at the top of `run_once_generate_waveform.py`.
2. Run it on any machine with numpy:
   ```bash
   python run_once_generate_waveform.py
   ```
3. The script prints a new `#define CLICK_LEN` line and `CLICK_WAV` array. Paste them into `click_player.ino`, replacing the existing block (lines 34–72).
4. If `CLICK_LEN` changed (i.e. you changed the click duration), also update `CLICK_WIDTH_S` in [RPi_main/config.py](../../RPi_main/config.py) and `CLICK_WIDTH_S` in [bmi_closed_loop/config.py](../../config.py) to match.
5. Reflash the ItsyBitsy as described above.
